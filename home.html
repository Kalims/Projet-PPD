Machine Learning avec TensorFlow.JS

Dans le cadre de notre formation par apprentissage en master MIAGE, nous avons eu l’opportunité de pouvoir gérer un projet dans son intégralité et ce en situation réelle. Plusieurs choix de sujets nous ont donc été soumis et nous avions choisi le sujet portant sur le Machine Learning avec TensorFlow.JS. TensorFlow est un outil open source d’apprentissage automatique développé par Google.

En savoir plus sur Tensorflow.js...

Les objectifs du projet sont :

Objectif n°1

Découverte des outils TensorFlow, tensorFlow.js et le langage python

Atteint

Objectif n°2

Familiarisation avec les méthodes basiques de Machine Learning

Atteint

Objectif n°3

Définition d’une interface riche avec Javascript, HTML, CSS et tensorFlow.js

Atteint

Objectif n°4

Création d'une application avec une classification d’images

Atteint

Il y aura une partie modification dynamique du modèle et de l’entraînement. Les paramètres pour l’entraînement seront :

  • Le type d’optimizer et sa valeur : paramètres du modèle
  • La taille du batch et les époques : paramètres de l’entraînement
  • Les images obtenues via la webcam : paramètres de l’entraînement

L’application sera disponible en ligne, pour pouvoir faire participer des gens. L’échantillon utilisé pour nos tests sera à l’échelle de notre classe (25 élèves). Concernant le stockage, il y a deux options :

  • On stocke les images brut une à une ;
  • On stocke le modèle avec les nouvelles données.

Bilan des fonctionnalités :

  • Utilisation de Tensorflow js avec le modèle MobileNet

    Exploitation de la bibliothèque Javascript MobileNet dans Tensorflow via l'utilisation des réseaux de neurones convolutionnels.

  • Sauvegarde des images prises par rapport à la webcam

    L'image est d'abord récupérée en format "canvas" via la fonction webcam de HTML puis est envoyée au back-end PHP en base64. Les images sont stockées par classes (répertoires) dans un nommage particulier afin d'être reconnues.

  • Entraîner un modèle à partir des images stockées en base

    Le format d'image attendu par MobileNet est en "canvas", il faut donc reconvertir les images en base64 du back en "canvas" afin d'utiliser les fonctions de convolutions, ReLU (Rectified Linear Unit), Pooling, Flatten, Fully connected, Softmax.

  • Paramétrages possibles du modèle

    Plusieurs paramètres du modèle sont personnalisables via l'interface du site web : le type d'optimizer, le taux d'apprentissage, le batch size et les epochs.

  • Résultats de l’entraînement sous forme de graphique de suivi

    À chaque entrainement du modèle, le graphique de la perte (loss) et le graphique de la précision (accuracy) sont créés. Ils se mettent à jour en temps réel.

  • Tableau des comparaisons entre résultats attendus et résultats prédits

    Génère un tableau qui compare chaque résultats attendus avec les résultats prédits par le modèle.

  • Prédiction de l’appartenance des visages

    Met à jour un tableau de toutes les classes détectées par la webcam. Affiche également le nom de la personne actuellement détectée.
